# Правила для ИИ-ассистентов (Claude Code / Cursor / др.)

## ⛔️ ЗАЩИЩЁННЫЕ ФАЙЛЫ — НЕ ТРОГАТЬ

Эти файлы относятся к аналитике и подтверждению прав в вебмастерах.
Их НЕЛЬЗЯ удалять, перемещать, переименовывать или «чистить», даже если они
кажутся лишними, неиспользуемыми или мешают рефакторингу.

1. **`frontend/js/counters.js`** — единый контейнер счётчиков (Яндекс.Метрика,
   Google Tag Manager, GA4) и мета-тегов. Логику не менять. Содержимое CONFIG
   внутри файла редактирует только владелец вручную.

2. **Строка подключения контейнера** во всех HTML-страницах `frontend/*.html`:

   ```html
   <script src="js/counters.js?v=1" data-counters></script>
   ```

   Не удалять эту строку при правках страниц. Если создаёшь новую HTML-страницу
   в `frontend/`, ОБЯЗАТЕЛЬНО добавь эту строку перед `</head>`.

3. **Файлы подтверждения прав вебмастеров** в корне `frontend/` —
   `google*.html`, `yandex_*.html`, `*verification*` и подобные. Не удалять.

4. **Мета-тег подтверждения прав** в `<head>` всех `frontend/*.html`:

   ```html
   <meta name="google-site-verification" content="..." />
   ```

   Не удалять. Роботы поисковиков не исполняют JS, поэтому этот тег обязан
   оставаться в самом HTML. Если создаёшь новую страницу — добавь его в `<head>`.

## Если просят «почистить» или «удалить лишнее»
Перечисленное выше из удаления исключать. При сомнении — спросить владельца.

# 🕯️ Память — Мемориальный сайт-летопись

## О проекте

«Память» — веб-платформа для создания мемориальных страниц об ушедших людях. Позволяет хранить биографии, фотографии, воспоминания близких, строить родословное древо и вести хронику рода.

## Принцип документации

> **Правило**: Единый документ (`docs/README.md`) описывает проект целиком. Каждая папка содержит свой локальный `README.md` с деталями реализации этой части.

Структура документации:
- `docs/README.md` — этот файл, общий обзор
- `docs/MIGRATION.md` — план миграции v1 → v2
- `docs/API.md` — справочник API endpoints
- `server/README.md` — документация бэкенда
- `server/docs/DATABASE.md` — схема БД (Prisma v2)
- `js/README.md` — документация фронтенда

## Архитектура

### Текущая версия (v1) — SQLite + vanilla JS

```
site-full_test1/
├── index.html              — Главная страница
├── memory.html             — Каталог страниц памяти
├── person.html             — Персональная страница
├── family-tree.html        — Родословное древо
├── timeline.html           — Летопись семьи
├── js/                     — Фронтенд-логика (vanilla JS)
├── styles/                 — CSS стили
├── server/                 — Node.js бэкенд (чистый, без фреймворков)
│   ├── prisma/             — Prisma-схема (целевая v2)
│   ├── docs/               — Документация БД
│   └── data/               — SQLite файл (v1)
```

### Целевая версия (v2) — PostgreSQL + Prisma + Telegram-бот

Миграция с SQLite на PostgreSQL. Авторизация через Telegram. Структурированные контент-блоки вместо одного текстового поля bio.

## Технологии

### Фронтенд
- Vanilla JavaScript (ES6+)
- CSS (без препроцессоров)
- Google Fonts (Cormorant Garamond, Playfair Display, Inter)
- Canvas (частицы)
- SVG (древо, лиана памяти)

### Бэкенд (v1 — текущий)
- Node.js 24+ (чистый, без Express)
- SQLite через `node:sqlite` (встроенный модуль)
- JWT (HS256) — собственная реализация
- PBKDF2-SHA512 — хеширование паролей
- Multipart parser — загрузка фото

### Бэкенд (v2 — целевой)
- Node.js + Prisma ORM
- PostgreSQL
- Авторизация через Telegram (telegramId)
- S3-совместимое хранилище для фото

---

## Запуск (v1)

```bash
cd server
node index.js
```

Сервер стартует на `http://localhost:3000`, раздаёт фронтенд автоматически.

---

## Источники данных

Фронтенд работает с тремя источниками (fallback-цепочка):

1. **API** (`/api/people/...`) — основной источник
2. **Telegram-бот** (`/bot-data/pages/{uuid}.json`) — страницы из бота
3. **js/data.js** — хардкод 18 людей (offline fallback)

## Статус миграции

| Компонент | v1 (текущий) | v2 (целевой) | Статус |
|-----------|-------------|-------------|--------|
| БД | SQLite | PostgreSQL + Prisma | Схема готова |
| Авторизация | email/password | Telegram ID | Не начато |
| Контент | bio (текст) | ContentBlock[] (зебра) | Фронтенд готов |
| Воспоминания | reviews (без модерации) | GuestMemory (isApproved) | Не начато |
| Фото | локальный upload | S3 | Не начато |
| Telegram-бот | Нет (только раздача файлов) | Полноценный бот | Не начато |

Подробности: `docs/MIGRATION.md`

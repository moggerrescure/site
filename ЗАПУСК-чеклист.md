## ⚠️ Требует вашего действия ДО запуска

**1. Заполнить секреты в `.env`** (без них часть функций не работает):
- `BOT_USERNAME` — username бота без @ (вход через Telegram).
- `ADMIN_CHAT_ID` — chat id для админ-алертов и поддержки.
- `AI_API_KEY` — иначе ИИ-ассистент для биографий не работает.
- `SMTP_HOST/USER/PASS` — иначе письма восстановления пароля не уходят.
- `S3_*` — опционально (без них файлы хранятся локально в `/uploads`).

**2. Применить новый пароль БД.** Если том `pgdata` ещё НЕ создавался (чистый
прод) — ничего не нужно, пароль применится при инициализации. Если база уже
существует:
```bash
docker compose exec db psql -U memorial -d memorial_site \
  -c "ALTER USER memorial WITH PASSWORD 'Ney2mrJKDdVtDWM4xiJPXZDIvTny';"
# либо, если данных ещё нет: docker compose down -v  (СОТРЁТ базу) и поднять заново
```
Для локальной разработки обновите пароль и в `backend/.env`.

**3. DNS + деплой.** Направить A/AAAA `qr-memory.by` и `www` на IP сервера,
открыть порты 80 и 443. Запускать прод БЕЗ override:
```bash
docker compose -f docker-compose.yml up -d --build
```
(override включает dev-режим и публикует Postgres на хост — в проде не использовать.)

**4. История git с дампом.** `db_dump.sql` остаётся в прошлых коммитах. Данные
похожи на демо (`admin@admin.local`), но если там были реальные ПДн — почистить
историю и форс-пушнуть (перезапишет историю, согласуйте с командой):
```bash
git filter-repo --path db_dump.sql --invert-paths
```

**5. Аналитика.** Вставить ID Яндекс.Метрики/GA4 в `frontend/js/counters.js`
(этот файл по правилам репо редактирует владелец вручную). Добавить сайт и
sitemap.xml в Google Search Console и Яндекс.Вебмастер.

**6. Перепроверить на своей машине** (песочница не смогла — нет сети к npm и Docker):
```bash
caddy validate --config frontend/Caddyfile --adapter caddyfile
docker compose -f docker-compose.yml config >/dev/null && echo OK
cd backend && npm audit --omit=dev        # уязвимости в зависимостях
node --check auth.js && node --check router.js
```

## 🎯 План: обогнать qr-memory.online

Их слабые места в выдаче: title — только бренд «QR-ПАМЯТЬ» (без ключевых слов),
описание общее, контента мало; домен `.online` (нет гео-сигнала для Беларуси).
Их сильная сторона — гравировка QR прямо на памятнике через партнёра-производителя.

Ваши козыри: ключевые `title`/`H1` уже есть, больше полезных страниц (древо,
летопись, FAQ со schema), `llms.txt` для ИИ-выдачи, домен `.by` (гео для РБ),
ИИ-ассистент. Порядок действий:

1. **Сначала закрыть блокеры 1–3** — без верного `SITE_URL`, HTTPS и индексации
   сайт вообще не попадёт в выдачу, обгонять будет нечем.
2. **Локальное SEO под Беларусь.** Гео-запросы в title/H1/тексты:
   «QR-код на памятник Минск/Брест/Гомель…», «электронный мемориал Беларусь».
   Под топ-города — отдельные посадочные страницы (не пустышки, с реальной пользой).
3. **Закрыть нишу конкурента.** Раз их фишка — гравировка на памятнике, сделайте
   у себя страницу «QR-табличка и гравировка на памятник» с ценами/гарантией/FAQ
   и партнёрами-гранитчиками в РБ.
4. **Контент под НЧ-запросы** (информационка → ссылки на коммерцию): «как сделать
   QR-код для могилы», «что писать в биографии», «как перенести фото на памятник».
5. **Schema по максимуму.** Добавить `Person`/`Memorial`-разметку на `/p/<slug>`
   (имя, годы жизни) — это даёт расширенные сниппеты по запросам с именами.
6. **Ссылки с белорусских площадок** (каталоги ритуальных услуг, гранитные
   мастерские, локальные СМИ/форумы) — основной фактор обгона в нише.
7. **GEO/ИИ-выдача.** `llms.txt` уже есть; добавляйте конкретные факты, цифры,
   FAQ — это повышает шанс цитирования в AI Overviews/ChatGPT/Perplexity.
8. **Замерить Core Web Vitals** главной (тяжёлый Three.js + 6 шрифтов): PageSpeed
   Insights → оптимизировать LCP (lazy-load 3D, `font-display: swap`, меньше шрифтов).
9. Подключить съём позиций по целевым запросам и сверяться с конкурентом раз в неделю.

# 09. Чек-листы / Checklists

Готовые списки для прогона. Печатай как есть и отмечай статус каждого пункта
(✅ ок / ⚠️ исправить / ❌ критично). Глубина по каждому пункту — в `01`–`08`.

## A. Запуск нового сайта с нуля (порядок)

1. [ ] Оценка бюджета/сроков + анализ конкурентов (referring domains) — `01`
2. [ ] Семантическое ядро (СЯ) — `02`
3. [ ] Кластеризация запросов по страницам — `02`
4. [ ] Title + H1 на каждый кластер (уникальный title, ключи в начало, гео) — `04`
5. [ ] ЧПУ-структура (≤3 уровня), плоский `/products/` для магазина — `03`
6. [ ] Description + Schema + favicon — `04`
7. [ ] robots.txt (на разработке `Disallow: /`, снять при запуске) + noindex служебных — `05`
8. [ ] sitemap.xml (чистый) — `05`
9. [ ] Google Search Console + отправка ключевых URL на индексацию — `05`
10. [ ] GA4 + события/конверсии — `05`
11. [ ] Выбор продвигаемых страниц (приоритет деньгам) + съём позиций — `01`,`05`
12. [ ] План ссылок: 30/70, типы анкоров — `07`
13. [ ] Закупка + бесплатные ссылки (темп ~15/мес для нового) — `07`
14. [ ] SEO-тексты (H2 под остатки ключей, польза, фото/цифры) — `06`
15. [ ] Перелинковка/PageRank: вес на денежные страницы — `07`
16. [ ] НЧ/информационные статьи — `06`
17. [ ] Скорость/Core Web Vitals, картинки — `05`
18. [ ] GEO-слой по странице — раздел F ниже
19. [ ] Контроль: позиции, индекс, поведенческие — итерации

## B. Технический SEO-аудит существующего сайта

Доступ/индексация:
- [ ] robots.txt без лишних запретов и без `Disallow: /`
- [ ] noindex на служебных, снят с продвигаемых
- [ ] sitemap.xml чистый и добавлен в GSC
- [ ] сайт в индексе (`site:домен`), ключевые URL проверены в GSC
- [ ] одна каноничная версия (http/https, www) + 301 + `rel=canonical`

Структура/URL:
- [ ] продвигаемые страницы ≤ 3-го уровня вложенности
- [ ] ЧПУ (читаемые URL, без `?id=&sort=`)
- [ ] любая смена URL/CMS — через 301; нет битых ссылок и цепочек редиректов

On-page:
- [ ] один `<title>` + один `<h1>` на странице
- [ ] title уникален во всём интернете (бренд-добавка)
- [ ] важные ключи в начале title; гео-добавка где нужно
- [ ] description заполнен; Schema заполнена; favicon выделяющийся
- [ ] H2 содержат продвигаемые запросы

Здоровье:
- [ ] Core Web Vitals: LCP ≤2.5s, INP ≤200ms, CLS ≤0.1
- [ ] адаптив / mobile-first, одна версия
- [ ] картинки: alt, сжатие, имена файлов
- [ ] HTTPS; нет Manual Actions и проблем безопасности в GSC

Аналитика:
- [ ] GA4 + конверсии настроены
- [ ] GSC подключён (покрытие, запросы)
- [ ] съём позиций по продвигаемым запросам

## C. On-page по одной странице

- [ ] Кластер запросов определён, интент проверен (коммерч./информац.) — `02`
- [ ] title (уникальный, ключи в начало, гео, ~70 симв.) — `04`
- [ ] h1 (один, уникален на сайте; можно отличать от title) — `04`
- [ ] H2 под оставшиеся ключи; H3 для логики — `06`
- [ ] description (преимущества + CTA + цифры/символы, ~150–160) — `04`
- [ ] Schema (Product/FAQPage/Article/Org) — `04`,`08`
- [ ] полезный текст для людей (фото, цифры, видео; без «воды») — `06`
- [ ] форма захвата/конверсия на странице — `01`,`07`
- [ ] внутренние ссылки на страницу (вес), мин. исходящих с денежной — `07`
- [ ] ответ на главный вопрос в первых ~50–200 словах (GEO) — `08`

## D. Ссылочный профиль

- [ ] распределение 30% главная / 70% целевые (или 30/35/35 при двух) — `07`
- [ ] смесь анкоров: безанкорные + коммерческие + некоммерческие — `07`
- [ ] темп набора без резких всплесков (новый сайт ~15/мес) — `01`,`07`
- [ ] метрики доноров (DR/CF/TF, трафик, тематика, регион) — `07`
- [ ] приёмка вебмастера: уникальный текст, правильная ссылка/анкор, индексация — `07`
- [ ] бесплатные: отзывы, каталоги, форумы, соцсети, GBP — `07`

## E. Перелинковка / внутренний PageRank

- [ ] вся «мощь» ведёт на зарабатывающие страницы — `07`
- [ ] нет «висячих узлов» (получают вес, не отдают) — `07`
- [ ] «О нас»/«Контакты» не съедают вес (якорь/возврат веса) — `07`
- [ ] блоки «Рекомендуем» ведут на нужные страницы — `07`
- [ ] неактуальные страницы не удалены, а ссылаются на актуальные — `07`
- [ ] (опц.) Netpeak Spider → расчёт внутреннего PageRank — `07`

## F. GEO / ИИ-выдача

- [ ] ИИ-краулеры не заблокированы (GPTBot, ClaudeBot, PerplexityBot, OAI-SearchBot, Google-Extended) — `08`
- [ ] ключевой контент в server-side HTML — `08`
- [ ] ответ на главный вопрос в первых ~50–200 словах — `08`
- [ ] самодостаточные абзацы; конкретные цифры/факты/сущности — `08`
- [ ] заголовки-вопросы + FAQ (дословные вопросы в Schema) — `06`,`08`
- [ ] списки/таблицы/листиклы под коммерческие кластеры — `08`
- [ ] свежесть: регулярные обновления + видимые даты — `08`
- [ ] уникальные данные/экспертиза; явный автор + E-E-A-T — `08`
- [ ] релевантные изображения/видео — `05`,`08`
- [ ] llms.txt — опционально, без завышенных ожиданий — `08`
- [ ] измерение: ручные проверки в ИИ + реферальный ИИ-трафик в GA4 — `08`

---

### Принцип отчёта по аудиту
Для каждого пункта: статус (✅/⚠️/❌) → что нашли → конкретная рекомендация
(как исправить). Порядок: критичное (индексация, редиректы, доступ) → важное
(релевантность, ссылки) → улучшения (скорость, GEO-нюансы). Каждую рекомендацию
привязывай к пользе/деньгам, а не к «потому что так в SEO».

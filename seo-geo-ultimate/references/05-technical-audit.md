<a name="robots"></a>
## robots.txt

Лежит **в корне** (`site.com/robots.txt`) — это первое, что читает поисковый бот.
Открыть можно у любого сайта (`/robots.txt`).

Базовые директивы:
- `User-agent: *` — обращение ко всем ботам (можно адресно: `Googlebot`, `Yandex`…).
- `Disallow: /path` — запрет обхода/индексации раздела.
- `Allow: /path` — явное разрешение (нужно, чтобы открыть что-то внутри
  запрещённой папки).
- `Sitemap: https://site.com/sitemap.xml` — указать карту сайта.

Принципы:
- **Не копируй чужой robots.txt** — он бесполезен без понимания. Закрывай только
  **своё** служебное: сортировки (`*sort=*`), результаты поиска, UTM-страницы,
  админку (`/wp-admin/` с `Allow: /wp-admin/admin-ajax.php`).
- Стандартный robots для WordPress на старте достаточно; усложняй по мере роста.
- **На время разработки нового сайта** закрой его целиком:
  ```
  User-agent: *
  Disallow: /
  ```
  После запуска — **убери** этот запрет (запиши в чек-лист, чтобы не забыть!).
- robots.txt — это запрет **обхода**, не строгая гарантия отсутствия в индексе:
  иногда приходит «проиндексировано, несмотря на запрет в robots.txt». Чтобы
  страницы **точно** не было в выдаче — используй `noindex` (ниже).
- Если CMS генерирует robots.txt динамически, твой физический файл в корне имеет
  приоритет.

<a name="noindex"></a>
## noindex (мета-тег robots / X-Robots-Tag)

Когда не хочешь «светить» список всех страниц в robots.txt — закрывай конкретную
страницу прямо в её коде:
```html
<meta name="robots" content="noindex, follow">
```
или заголовком ответа `X-Robots-Tag: noindex` (для не-HTML).

- Ставь на служебные страницы: «Спасибо»/thank-you, корзина, личный кабинет,
  фильтры-дубли.
- **robots.txt vs noindex:** массовые динамические страницы (сортировки,
  результаты поиска), которых физически нет до запроса, удобнее закрывать в
  **robots.txt** (на них не поставишь meta). Статические служебные страницы —
  через **noindex**.
- Важно: если страница закрыта в `robots.txt`, бот может **не увидеть** `noindex`
  на ней (он туда не зайдёт). Не закрывай одно и то же двумя способами сразу.

<a name="sitemap"></a>
## sitemap.xml (карта сайта)

- Назначение: подсказать поисковику все нужные URL (особенно полезно для больших
  сайтов — «вывести 10 000 страниц сразу»). CMS обычно генерирует её сама.
- **Почисти карту:** оставь только **продвигаемые/индексируемые** страницы.
  Убери служебные и (по желанию) изображения, если для них отдельная логика.
- Страницы, исключённые из sitemap, при необходимости дополнительно закрой
  (`noindex`), если они не должны быть в индексе, — отсутствие в карте само по
  себе индексацию не запрещает.
- **Добавь sitemap в Google Search Console** (раздел «Файлы Sitemap»).

<a name="gsc"></a>
## Google Search Console (GSC)

Бесплатный «вебмастер» от Google — обязателен.

Добавление ресурса:
- **Доменный ресурс** (Domain) — покрывает все поддомены и протоколы; верификация
  через DNS-запись.
- **Ресурс с префиксом** (URL-prefix) — конкретный `https://site.com/`;
  верификация HTML-файлом/тегом/GA/GTM.

Дальше:
- Сайт должен **редиректить на одну выбранную версию** адреса (см. `03`).
- Отправляй ключевые страницы на индексацию через **«Проверка URL» → Запросить
  индексирование**.
- Удаляй ненужные страницы из индекса («Удаления»).
- Следи за разделами: **Покрытие/Индексирование**, **Эффективность** (запросы,
  CTR, позиции), **Core Web Vitals**, **Меры, принятые вручную** (Manual Actions),
  **Проблемы безопасности**.
- При переезде на новый домен используй инструмент **«Переезд сайта»**.

<a name="индексация"></a>
## Индексация: проверка и управление

- Проверка наличия: `site:site.com` в Google; точечно — «Проверка URL» в GSC.
- Если важные страницы не в индексе: проверь `noindex`, robots.txt, канониклы,
  редиректы, доступность (200 OK), качество/уникальность контента, внутренние
  ссылки на страницу.
- Дубли: задай `rel=canonical` на основную версию; убери параметрические дубли.

<a name="ga4"></a>
## Google Analytics 4 + конверсии

- Создай аккаунт GA4, поток данных (data stream), получи **Measurement ID**
  (`G-XXXX`).
- Установка: код **gtag.js** прямо в `<head>` **или** через **Google Tag Manager
  (GTM)** (предпочтительно — гибче).
- Настрой **события** и **конверсии** (отправка формы, звонок, заказ) — через GTM
  или встроенными событиями. Без целей ты не измеришь, окупается ли SEO.
- Проверь работу счётчика (Realtime).

<a name="позиции"></a>
## Съём позиций (rank tracking)

Нужно отслеживать, насколько эффективно идёт работа.
- Вручную: вбивать продвигаемые запросы **в инкогнито** (история искажает выдачу).
- Автоматически: Serpstat/аналоги — создать проект, задать **запросы**, **регион**
  и **устройство** (desktop/mobile), частоту съёма; смотреть историю позиций и
  конкурентов.
- Ориентир по деньгам: реальные заявки обычно начинаются с **ТОП-3…ТОП-5**; на
  10-й позиции трафика мало. Цель — первый экран.

<a name="скорость"></a>
## Скорость загрузки и Core Web Vitals

- Инструмент: **PageSpeed Insights** (и отчёт CWV в GSC).
- Метрики **Core Web Vitals**: **LCP** (загрузка основного контента, ≤2.5s),
  **INP** (отзывчивость на ввод, ≤200ms), **CLS** (визуальная стабильность, ≤0.1).
- Влияние: скорость — фактор ранжирования, но **не решающий**. Сначала
  релевантность и ссылки; скорость даёт прирост «на грани», когда конкуренты
  равны. Медленный сайт также режет конверсию и поведенческие.
- Что обычно чинят: сжатие и современные форматы картинок (WebP/AVIF), lazy-load,
  кэш и CDN, минификация CSS/JS, отложенная загрузка скриптов, нормальный хостинг.

<a name="картинки"></a>
## Картинки (image SEO)

- **alt** — описание изображения (и для доступности, и для Google Картинок);
  **title** по необходимости. Прописывай там, где это осмысленно (товары, важные
  фото), не ради «галочки» на каждом декоративном элементе.
- Нормальные **имена файлов** (транслит/англ., по смыслу), сжатие, корректные
  размеры, отдавай через современные форматы.
- ChatGPT/ИИ умеют описывать изображения — можно ускорять генерацию alt, но
  проверяй осмысленность.
- Картинки и видео усиливают и обычную выдачу, и ИИ-обзоры (`08`).

<a name="чеклист"></a>
## Чек-лист технического аудита (быстрый прогон)

Доступ/индексация: robots.txt без лишних запретов и без `Disallow: /` после
запуска · noindex на служебных, снят с продвигаемых · sitemap.xml чистый и в GSC
· сайт в индексе (`site:`) · одна каноничная версия + 301 + canonical.

Структура/URL: продвигаемое ≤ 3-го уровня · ЧПУ · смена URL только через 301 ·
нет битых ссылок и цепочек редиректов.

On-page: один title + один h1 · title уникален (бренд) · ключи в начало · гео-
добавка · description заполнен · Schema заполнена · favicon выделяющийся.

Здоровье: Core Web Vitals (LCP/INP/CLS) · адаптив/mobile-first · HTTPS · картинки
(alt, вес) · нет Manual Actions и проблем безопасности в GSC.

Аналитика: GA4 + конверсии · GSC подключён · съём позиций настроен.

GEO-слой (`08`): ИИ-краулеры не заблокированы · ответ в первых ~50–200 словах ·
цифры/факты · автор/E-E-A-T · FAQ-схема.

Формат вывода: по каждому пункту — ✅/⚠️/❌, находка, конкретная рекомендация.
Сначала критичное (индексация, редиректы) → важное → улучшения.

<a name="vs"></a>
## Визуальная структура ≠ структура по URL

Хлебные крошки (breadcrumbs) и визуальная навигация — это **логическая**
структура для пользователя. Реальная **структура по URL** может быть совсем
другой, и это нормально — крупные сайты так и работают. Не путай их.

Что важно для SEO — именно **URL-структура** и **уровень вложенности**. Сделай её
по единой понятной логике один раз — тогда визуальную навигацию сможешь менять
как угодно потом.

<a name="уровень"></a>
## Главное правило: 3-й уровень вложенности

Уровень считается по URL:
- `site.com/` — 1-й уровень (главная).
- `site.com/category/` — 2-й уровень.
- `site.com/category/item/` — 3-й уровень.

**Продвигаемые страницы не должны быть глубже 3-го уровня. Идеал — 2-й.**
Чем ближе страница к главной, тем больше внутреннего веса (link weight) она
получает (подробно про вес — `07`).

<a name="оптимальная"></a>
## Оптимальная структура

**Небольшой сайт услуг:** главная → все услуги сразу на 2-м уровне.
```
site.com/
site.com/uteplenie-domov/
site.com/uteplenie-kvartir/
site.com/uteplenie-fasada-penoplastom/
```
Даже если услуг сотня — держи их на 2-м уровне, не углубляй без нужды.

**Интернет-магазин:** главная → категории (2-й уровень) → товары в общем
разделе `/products/` (3-й уровень).
```
site.com/category-pylesosy/
site.com/category-televizory/
site.com/products/pylesos-bosch-123/
```
Почему товары кладут в плоский `/products/`, а не внутрь категории: **один товар
может быть в нескольких категориях** (пылесос-робот — и в «пылесосы», и в «роботы-
пылесосы»). Привязка товара к категориям делается тегами/метками в CMS, а URL
остаётся один. Слово `products` в URL может даже не фигурировать визуально — это ок.

Поддомены (subdomains) на старте не нужны — не усложняй.

<a name="чпу"></a>
## ЧПУ — человекопонятные URL (clean URLs)

- Читаемые слаги: `/uteplenie-fasada/`, `/vacuum-cleaners/`.
- **Без** `?id=123&sort=abc`, без абракадабры. Такие URL выглядят и
  индексируются хуже и неудобны (например, для аудиторий ремаркетинга).
- Транслитерация — годится любая (хоть «официальная», хоть своя), лишь бы было
  понятно людям. На продвижение конкретный вариант транслита не влияет.
- CMS обычно сами переводят заголовок в нормальный слаг.

<a name="редиректы"></a>
## Редиректы 301 — критично (как «просрать» позиции)

Для поисковика **смена URL = удаление старой страницы и создание новой**. Связь
между ними он не знает, пока ты не поставишь редирект. Поэтому:

> Любое изменение URL существующей (уже проиндексированной) страницы — только
> через **301-редирект** старый_URL → новый_URL.

Это касается:
- смены слага/структуры;
- **переезда на новый движок (CMS)** — классическая катастрофа: «перенесли»
  товары, но URL поменялись, для поисковика все страницы исчезли, сайт «пропал»;
- даже мелочи: другая буква/транслит — это уже другая страница.

301 («Moved Permanently») — для постоянного переноса (наш случай). Бывают и другие
типы, но для переноса страницы нужен именно 301.

**Если делаешь сайт с нуля** — редиректы обычно не нужны (страниц в индексе ещё
нет): сразу заложи правильную ЧПУ-структуру, и переносить ничего не придётся.

Дополнительно через редиректы решают: склейку `www`/без `www`, `http`→`https`,
перенос на новый домен (`.com.ua` → `.ua`) — часто это одна строка кода для
всего домена, если структура не меняется.

<a name="htaccess"></a>
## htaccess: как сделать 301 (Apache)

Файл `.htaccess` лежит в корне (у него нет имени, только расширение). Он
описывает, как работают ссылки на сайте (для WordPress, например, обеспечивает
человекопонятные URL). Редиректы пишутся так:

```apache
# одиночный редирект
Redirect 301 /old-page/ https://site.com/new-page/

# с www на без www + https (пример)
RewriteEngine On
RewriteCond %{HTTP_HOST} ^www\.(.+)$ [NC]
RewriteRule ^ https://%1%{REQUEST_URI} [R=301,L]
```

**Массовые редиректы через таблицу.** Если страниц много, собери в Excel/Sheets
3 колонки: `старый_URL | новый_URL | строка_301` и сгенерируй строки формулой
сцепления:
```
="Redirect 301 "&" "&A2&" "&B2
```
Скопируй готовый столбец в `.htaccess`. На позиции это не влияет негативно,
для пользователя редирект мгновенный.

> На разных серверах синтаксис разный (Nginx — не `.htaccess`, а конфиг с
> `rewrite`/`return 301`). Уточни стек; при необходимости попроси сгенерировать
> правило под нужный сервер.

<a name="главная"></a>
## Не продвигай главную страницу

С точки зрения SEO продвигать главную можно, но **не стоит**:
- бизнес со временем меняется (добавил направление — главная теперь про другое);
- на главную **нельзя поставить редирект** (весь трафик домена идёт на неё).

Поэтому ключевые услуги — на отдельных страницах 2-го уровня. На главной — максимум
общий запрос ниши и бренд, без фанатизма. Тогда структуру и бизнес можно
безопасно развивать. (Главная при этом получает много ссылочного веса и
«проливает» его на вложенные страницы — см. `07`.)

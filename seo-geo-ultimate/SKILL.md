# SEO + GEO: Ультимативный скилл / Ultimate Skill

Это рабочая методичка для продвижения сайта в ТОП Google **и** для попадания в
ИИ-выдачу (GEO). Основа — авторский курс по SEO (32 урока, методика «10% усилий →
90% результата»), дополненный актуальными рекомендациями по GEO из открытых
источников (Google Search Central, Search Engine Land и др. — см.
`references/08-geo-ai-search.md`).

Скилл **универсальный**: подходит любому сайту и любой CMS (WordPress, самопис,
конструкторы, интернет-магазины).

---

## 0. Философия / Core philosophy (читать первым)

Эти принципы важнее любой отдельной техники. Если действие им противоречит —
не делай его.

1. **SEO ради денег, а не ради SEO.** Цель — продажи и заявки (leads), а не
   «трафик ради трафика». Сначала продвигаем то, что приносит больше денег.
   Много нерелевантного трафика (например, информационка вместо коммерции) —
   это не результат.
2. **Human to human. Здравый смысл, а не погоня за апдейтами.** Всё, что делает
   Google, в итоге служит удобству людей. Кто подстраивается под алгоритмы и
   тренды — теряет позиции на каждом апдейте; кто делает по здравому смыслу —
   стабильно в ТОПе. **Это же — выигрышная стратегия для GEO** (см. §8): Google
   официально говорит, что оптимизация под ИИ — это та же забота о пользователе
   и качественный, не-«ширпотребный» контент.
3. **SEO — это вдолгую.** Нужен результат быстро → это реклама (Google Ads).
   SEO даёт более дешёвый и «горячий» трафик, но месяцами. Новый сайт может
   полгода–год сидеть в «песочнице» (sandbox).
4. **Мы помогаем системе понять контент, а не обманываем её.** Правильные
   title/H1/H2, структура и микроразметка — это сигналы релевантности, чтобы нас
   корректно подняли, а не «накрутка».
5. **Не плоди мусор.** Лучше не делать SEO-текст вообще, чем налить
   водянистый «текст ради ключей». Лишних страниц-пустышек быть не должно.

## 2. Мастер-воркфлоу / Master workflow (порядок имеет значение)

Это каноничный порядок из курса. Не перескакивай: каждый шаг опирается на
предыдущий. Подробности — в соответствующем reference.

1. **Оценка бюджета и сроков** + **анализ конкурентов** → `01`.
   Посмотреть ТОП по целевым запросам, посчитать среднее число ссылающихся
   доменов (referring domains) у конкурентов = сколько ссылок нужно нам.
2. **Семантическое ядро (СЯ)** → `02`. Собрать запросы (это фундамент, не
   на скорую руку).
3. **Кластеризация** → `02`. Сгруппировать запросы по страницам (ручная или
   автоматическая — у каждой свои плюсы).
4. **Title и H1** → `04`. Самое основное: правильные title/H1 уже выводят в
   ТОП по регионам. Запросы, не вошедшие в title/H1 — отложить под H2 (шаг 9).
5. **Структура сайта, ЧПУ, редиректы** → `03`. Продвигаемые страницы — не
   глубже 3-го уровня вложенности; смена URL только через 301-редирект.
6. **Description + микроразметка + favicon** → `04`. Задача — поднять CTR
   сниппета.
7. **Технический минимум**: robots.txt / noindex, sitemap.xml, добавить сайт
   в **Google Search Console**, поставить **GA4** и цели/конверсии → `05`.
8. **Выбор продвигаемых страниц** (приоритет — деньги) + подключить съём
   позиций (rank tracking) → `01` и `05`.
9. **Ссылки**: распределение по страницам (≈30% на главную / 70% на целевые),
   анкоры (безанкорные / коммерческие / некоммерческие), затем закупка и
   бесплатные ссылки → `07`.
10. **SEO-тексты на страницы**: H2/H3 под оставшиеся запросы, полезный контент,
    фото/видео/цифры → `06`.
11. **Перелинковка / внутренний PageRank**: сконцентрировать вес на
    зарабатывающих страницах → `07`.
12. **НЧ-статьи** (низкочастотка) и информационка — в самом конце, отдельными
    страницами → `06`.
13. **Технические доработки** (скорость, картинки) → `05`.
14. **Контроль**: позиции, индексация, поведенческие; повторять итерации → `05`.

> Параллельно на всех текстовых шагах (4, 6, 10, 12) держи в голове **§8 GEO** —
> это не отдельный этап, а слой поверх обычного SEO.

## 3. Быстрый технический SEO-аудит / Fast technical audit (главный сценарий)

Если просят «проверь сайт» / «сделай SEO-аудит» — прогони этот чек-лист и выдай
отчёт. Глубина по каждому пункту — в `references/05-technical-audit.md`,
`03` и `04`. Полный печатный чек-лист — в `references/09-checklists.md`.

**A. Индексация и доступ**
- `robots.txt` в корне: не закрыто ли лишнее? нет ли `Disallow: /` после
  запуска? открыты ли нужные разделы?
- `noindex` (мета-тег / X-Robots-Tag): стоит на служебных (корзина, «спасибо»,
  сортировки, результаты поиска), снят с продвигаемых.
- `sitemap.xml`: есть, чистый (только продвигаемые страницы), добавлен в GSC.
- Сайт реально в индексе: `site:домен` + проверка ключевых URL в GSC.
- Одна каноничная версия адреса (http/https, www/без www, слэш) — остальные
  301-редиректят на неё; `rel=canonical` против дублей.

**B. Структура и URL**
- Продвигаемые страницы не глубже **3-го уровня** вложенности (идеал — 2-й).
- ЧПУ (человекопонятные URL): читаемые слаги, без `?id=...&sort=...`.
- Любая смена URL/CMS — через **301-редирект** старый→новый (иначе потеря
  позиций). Битых ссылок и цепочек редиректов нет.

**C. Релевантность (on-page)**
- Ровно **один** `<title>` и **один** `<h1>` на странице.
- `title` уникален во всём интернете (добавить бренд), `h1` уникален в пределах
  сайта; важные ключи — в начало; гео-добавка для региональных запросов.
- `description` заполнен (CTR), микроразметка/Schema заполнена по максимуму,
  favicon выделяющийся.
- Заголовки H2/H3 структурируют контент; продвигаемые запросы — в H2.

**D. Технич. здоровье**
- Скорость / **Core Web Vitals** (PageSpeed Insights): LCP, INP, CLS.
- Адаптив (mobile-first), одна версия (адаптив, а не отдельный m.-домен по
  возможности).
- Картинки: alt, сжатие, нормальные имена файлов.
- HTTPS, нет ошибок безопасности в GSC, нет ручных санкций (Manual Actions).

**E. Аналитика и контроль**
- GA4 стоит, события/конверсии настроены.
- GSC подключён, отслеживаются покрытие индекса и запросы.
- Подключён съём позиций по продвигаемым запросам.

**F. GEO-слой (см. §8)**
- ИИ-краулеры не заблокированы (если цель — цитируемость в ИИ).
- Ответ на главный вопрос страницы — в первых ~50–200 словах.
- Есть конкретные цифры/факты, автор/экспертность (E-E-A-T), FAQ-блоки.

Формат отчёта: для каждого пункта — статус (✅ ок / ⚠️ исправить / ❌ критично),
что именно нашли, и конкретная рекомендация. Сначала критичное (индексация,
редиректы), потом важное, потом улучшения.

## 4. Принципы вывода / How to respond

- Говори на языке пользователя; ключевые термины давай с английским
  эквивалентом при первом упоминании (title, referring domains, Core Web Vitals).
- Давай **конкретику и примеры**, а не общие слова. Где уместно — фрагменты кода
  (robots.txt, .htaccess, JSON-LD), таблицы, готовые шаблоны.
- Не выдумывай цифры конкурентов/позиции — их надо смотреть в реальных
  инструментах (Serpstat/аналоги, GSC). Объясняй, как проверить, а не угадывай.
- Привязывай каждую рекомендацию к деньгам/пользе, а не к «потому что так в SEO».
- При работе с GEO честно разделяй: что подтверждает Google официально, а что —
  наблюдения индустрии (особенно для ChatGPT/Perplexity). См. `08`.

## 5. Глоссарий-минимум / Mini-glossary

СЯ — семантическое ядро (semantic core). ЧПУ — человекопонятный URL (clean URL).
НЧ/СЧ/ВЧ — низко-/средне-/высокочастотные запросы (low/mid/high-frequency).
Анкор (anchor) — видимый текст ссылки. Referring domains — ссылающиеся домены.
Сниппет (snippet) — блок сайта в выдаче. GSC — Google Search Console.
GEO — Generative Engine Optimization; AEO — Answer Engine Optimization.
AI Overviews / AI Mode — ИИ-обзоры и ИИ-режим в Google.
E-E-A-T — Experience, Expertise, Authoritativeness, Trust.

Мы провели детальный интерактивный обход страниц локального сайта:
1. Главная страница (`/index.html`)
2. Страница семейного древа (`/family-tree.html`)
3. Профиль памяти (`/person.html`)
4. Каталог страниц памяти (`/memory.html`)
5. Страница FAQ (`/faq.html`)

Ниже собраны основные слабые места интерфейса с точки зрения плавности, производительности анимаций, отзывчивости и мобильной адаптации, а также конкретные предложения по их исправлению.

## Сводная таблица замечаний

| Before | After | Why |
| --- | --- | --- |
| `transition: all 0.35s cubic-bezier(0.4,0,0.2,1)` (в `tree.css`, `person.css`, `main.css`) | `transition: transform 0.2s cubic-bezier(0.23, 1, 0.32, 1), opacity 0.2s cubic-bezier(0.23, 1, 0.32, 1), background-color 0.2s cubic-bezier(0.23, 1, 0.32, 1);` | Избегайте анимации тяжелых свойств разметки (`layout`). Указание конкретных свойств исключает перерасчеты стилей для дочерних элементов и ускоряет рендеринг. |
| `--transition: 0.35s cubic-bezier(0.4, 0, 0.2, 1);` | `--transition: 0.2s cubic-bezier(0.23, 1, 0.32, 1);` | Длительность 350мс для базовых микро-взаимодействий (наведение на ссылки, чекбоксы, кнопки) кажется медленной и «тяжелой». Анимация должна завершаться быстрее 300мс. |
| `@keyframes treePageIn { 0% { transform: scale(0.55) translateY(60px); opacity: 0; } }` | `@keyframes treePageIn { 0% { transform: scale(0.95) translateY(16px); opacity: 0; } }` | Никакие крупные элементы в реальном мире не появляются из масштаба `0.55` без ощущения визуального диссонанса. Начинайте анимацию с `0.95` или `0.9` с прозрачностью. |
| `.btn:active { transform: translateY(0) scale(.99); }` | `.btn:active { transform: scale(0.97); }` | Сжатие кнопки на `1%` практически незаметно на глаз. Для физического тактильного отклика кнопки должны сжиматься сильнее (`0.95-0.98`) и мгновенно реагировать на нажатие. |
| Отсутствие состояния `:active` на интерактивных элементах (ссылках меню, карточках древа, точках карусели) | Добавление `transform: scale(0.97)` (или `scale(0.95)`) на селектор `:active` | Без `:active` интерфейс кажется «глухим» и неотзывчивым при нажатии на мобильных устройствах или десктопе. |
| `.tree-node:hover .tree-node__frame { transform: translateY(-5px) scale(1.04); }` (без проверки на тач-устройства) | Wrap in `@media (hover: hover) and (pointer: fine) { ... }` | На мобильных устройствах наведение «залипает» после нажатия пальцем, оставляя карточки приподнятыми в воздухе до тех пор, пока пользователь не кликнет в другое место. |
| Мгновенное удаление модального окна из DOM: `modal.remove()` | Добавление класса `.modal--leaving` с анимацией выхода, удаление после события `transitionend` | Резкое исчезновение модального окна без анимации закрытия выглядит как баг или обрывистый переход. |
| Резкое открытие/закрытие FAQ (`<details>` и `<summary>`) | CSS Grid transition (`grid-template-rows: 0fr -> 1fr`) на контент или JS accordion | Нативные `<details>` раскрываются мгновенно, что нарушает плавность интерфейса. |

## Детальный разбор слабых мест

### 1. Избыточное использование `transition: all`
**Где обнаружено:**
* В `main.css:302` и `polish.css:40` для кнопок `.btn`
* В `main.css:377` для `.feature-card`
* В `main.css:1239` для `.last-added .person-card`
* В `tree.css` для кнопок редактирования, легенды древа `.tree-legend__item`, элементов интерфейса `.tree-node-ctrl`, `.tree-modal__btn`, `.tree-add-btn`.
* В `person.css:262` и `person.css:540`

**Рекомендация:**
Заменить на явное перечисление анимируемых свойств, предпочтительно тех, что обрабатываются GPU (transform, opacity, filter):
```css
.btn {
  transition: transform 0.2s cubic-bezier(0.23, 1, 0.32, 1), 
              background-color 0.2s ease, 
              box-shadow 0.2s ease, 
              border-color 0.2s ease;
}
```

---

### 2. Медленная базовая длительность (`350ms`) и плавность
**Где обнаружено:**
* Переменная `--transition: 0.35s` в `main.css:27` используется для большинства ссылок навигации, иконок и фильтров.
* Анимации переключения веток древа (`treePageOut`/`treePageIn`) длятся `0.65s` (650мс), что заставляет пользователя ждать прогрузки.
* Переходы фильтрации на странице древа (при подсветке предков/потомков) имеют длительность `0.5s` на прозрачность и размытие.

**Рекомендация:**
Сократить длительность микро-взаимодействий до `150ms-200ms` и использовать более резкую кривую ускорения на выходе (`ease-out`), например, `cubic-bezier(0.16, 1, 0.3, 1)`:
```css
:root {
  --transition: 0.2s cubic-bezier(0.23, 1, 0.32, 1);
}
```

---

### 3. Слишком размашистые масштабирующие анимации (`scale`)
**Где обнаружено:**
* В `tree.css` анимации смены страниц `@keyframes treePageIn` и `@keyframes treePageOut` сжимают весь интерфейс дерева до `scale(0.55)`.
* Это приводит к резким скачкам текста и элементов, выглядит неестественно и перегружает рендеринг страницы.

**Рекомендация:**
Любые крупные контейнеры и страницы должны масштабироваться в пределах `0.95` - `1.0` (максимум `0.9`):
```css
@keyframes treePageIn {
  0% { transform: scale(0.95) translateY(16px); opacity: 0; }
  100% { transform: scale(1) translateY(0); opacity: 1; }
}
```

---

### 4. Отсутствие тактильного отклика (`:active`) при нажатии
**Где обнаружено:**
* Кнопки отправки форм, закрытия модальных окон (`.code-modal__submit:active`, `.ctm-submit:active`, кнопки в хедере `.btn--ghost`) либо вообще не имеют состояния `:active`, либо используют `transform: translateY(0)` / `scale(0.99)`.
* Карточки людей в каталоге (`.person-card`), ссылки навигации (`.nav__link`) и переключатели галереи (`.gallery-carousel__btn`, `.gallery-dot`) никак не реагируют на нажатие пальцем/мышкой в момент клика.

**Рекомендация:**
Для всех кликабельных элементов добавить легкий эффект сжатия:
```css
.btn:active,
.tree-node:active,
.gallery-carousel__btn:active,
.review-type-tab:active {
  transform: scale(0.97);
}
```

---

### 5. «Залипающие» эффекты наведения на мобильных устройствах
**Где обнаружено:**
* На мобильных устройствах ховер срабатывает при касании. Карточка дерева `.tree-node` приподнимается по `translateY(-5px)` и увеличивается.
* Поскольку события ухода указателя нет, карточка остается висеть в воздухе, пока пользователь не нажмет в другое место. Это портит впечатление от просмотра на смартфонах.
* Это же касается карточек родственников (`.person-card:hover`) и кнопок фильтров.

**Рекомендация:**
Оберенуть все эффекты смещения и масштабирования при наведении в медиа-запросы hover:
```css
@media (hover: hover) and (pointer: fine) {
  .tree-node:hover .tree-node__frame {
    transform: translateY(-5px) scale(1.04);
  }
}
```

### 6. Обрывистое закрытие модальных окон
**Где обнаружено:**
* Модальное окно авторизации (`#auth-overlay`), окно создания древа (`.ctm-close`), окно выбора связей исчезают мгновенно, так как JS вызывает `remove()` без ожидания анимации выхода. Вход при этом анимирован плавно.

**Рекомендация:**
Добавлять класс ухода `.leaving`, запускать CSS-переход прозрачности и масштаба, и удалять элемент по событию `transitionend`:
```js
function closeModal(modalEl) {
  modalEl.classList.add('leaving');
  modalEl.addEventListener('transitionend', () => {
    modalEl.remove();
  }, { once: true });
}
```
```css
.modal-overlay {
  transition: opacity 0.2s cubic-bezier(0.23, 1, 0.32, 1);
}
.modal-overlay.leaving {
  opacity: 0;
}
.modal-content {
  transition: transform 0.2s cubic-bezier(0.23, 1, 0.32, 1);
}
.modal-overlay.leaving .modal-content {
  transform: scale(0.95);
}
```

### 7. FAQ-аккордеон без анимации раскрытия
**Где обнаружено:**
* На странице `faq.html` раскрытие вопросов выполнено нативными `<details>` и `<summary>`. Текст ответов появляется мгновенно, создавая резкие скачки интерфейса.

**Рекомендация:**
Использовать CSS Grid для анимации высоты обертки контента от `0fr` до `1fr`:
```css
.faq-details[open] .faq-content-wrapper {
  grid-template-rows: 1fr;
}
.faq-content-wrapper {
  display: grid;
  grid-template-rows: 0fr;
  transition: grid-template-rows 250ms cubic-bezier(0.23, 1, 0.32, 1);
}
.faq-content {
  overflow: hidden;
}
```

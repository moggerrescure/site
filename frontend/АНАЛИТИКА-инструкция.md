# Аналитика и счётчики — как пользоваться контейнером

Все счётчики живут в одном файле — **`frontend/js/counters.js`** («контейнер»).
Он подключён ко всем страницам автоматически, в самом HTML кодов счётчиков нет,
поэтому правки сайта (в т.ч. автоматические, ИИ) их не затрагивают.

Файлы-участники:
- `frontend/js/counters.js` — сам контейнер (сюда вписываешь ID).
- строка `<script src="js/counters.js?v=1" data-counters></script>` — стоит перед
  `</head>` на каждой странице (трогать не нужно).
- `CLAUDE.md` и `AGENTS.md` в корне проекта — правило, запрещающее ИИ удалять
  контейнер и подключение.

---

## 1. Подключить счётчик (Метрика, GTM, GA4)

1. Открой `frontend/js/counters.js`.
2. Найди блок `CONFIG` вверху и впиши ID в кавычки. Примеры:

   ```js
   yandexMetrikaId: "98765432",     // номер счётчика Яндекс.Метрики
   gtmId:           "GTM-ABC1234",  // ID контейнера Google Tag Manager
   ga4Id:           "G-XXXXXXXXXX", // ID Google Analytics 4 (если нужен отдельно)
   ```

   Пустая строка `""` = счётчик выключен. Несколько можно включить одновременно.
3. Сохрани файл.
4. **Сбрось кэш**: на сервере JS кэшируется надолго. После изменения ID подними
   версию в подключении на страницах: замени `counters.js?v=1` на `?v=2`
   (потом `?v=3` и т.д.). Проще всего — глобальная замена по `frontend/*.html`.
   Без этого браузеры могут отдавать старую версию файла.

> GA4 обычно настраивают **внутри GTM** — тогда `ga4Id` оставь пустым, а GA4
> добавь тегом в интерфейсе Google Tag Manager.

## 2. Подтвердить права в вебмастере (Google / Яндекс)

⚠️ Важно: поисковые роботы **не выполняют JavaScript**. Поэтому мета-тег
верификации, добавленный через `counters.js`, они **не увидят** — для
подтверждения прав он не годится. Используй способы ниже (правка кода сайта не
нужна, ИИ их не тронет).

### Google Search Console

**Способ «HTML-файл» (рекомендую):**
1. В Search Console выбери подтверждение → «Загрузка файла HTML».
2. Скачай файл вида `google1a2b3c4d5e.html`.
3. Положи его **в папку `frontend/`** рядом с `index.html`.
4. Задеплой сайт. Файл откроется по адресу `https://qr-memory.by/google1a2b3c4d5e.html`.
5. Нажми «Подтвердить». Файл не удаляй — права слетят.

**Альтернатива — DNS:** добавь TXT-запись у регистратора домена (способ
«Поставщик доменных имён»). Тоже без правки кода.

### Яндекс.Вебмастер

**Способ «через Яндекс.Метрику» (самый простой):** если в `counters.js` включена
Метрика и счётчик в том же аккаунте Яндекса — в Вебмастере выбери подтверждение
через Метрику. Файлы и код не нужны.

**Способ «HTML-файл»:** скачай `yandex_xxxx.html`, положи в `frontend/`, задеплой,
подтверди. Откроется по `https://qr-memory.by/yandex_xxxx.html`.

**Альтернатива — DNS TXT.**

- [ ] Проверил в браузере: счётчик грузится (DevTools → Network: `mc.yandex.ru` /

- [ ] Положил файл верификации в `frontend/` (или включил DNS / привязку Метрики)

При создании новой HTML-страницы добавь перед `</head>`:
`<script src="js/counters.js?v=1" data-counters></script>`
